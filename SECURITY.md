# Política de Segurança

## Reportando Vulnerabilidades

Se você descobrir uma vulnerabilidade de segurança no ParamStrike, **não abra uma issue pública**. Em vez disso, envie um email para o mantenedor.

### Como Reportar

1. **Não divulgue publicamente** a vulnerabilidade
2. **Envie um email** descrevendo:
   - O tipo de vulnerabilidade
   - A localização do código afetado
   - O impacto potencial
   - (Opcional) Uma sugestão de correção

3. **Aguarde resposta** dentro de 48 horas

### O Que Esperar

- Você receberá uma confirmação de recebimento
- Um timeline será estabelecido para corrigir o problema
- Você será creditado na comunicação de segurança (se desejar)

## Divulgação Responsável

Seguimos o modelo de divulgação responsável:

1. **Relatório Privado** → Você nos avisa em particular
2. **Confirmação** → Respondemos em 48h
3. **Patch** → Corrigiremos o problema em sigilo
4. **Release Seguro** → Publicamos a correção
5. **Crédito Público** → Você é creditado (opcional)

## Escopo

Esta política cobre:
- ✅ Vulnerabilidades no código do ParamStrike
- ✅ Vulnerabilidades em dependências diretas
- ✅ Problemas que afetam a segurança do usuário

Fora do escopo:
- ❌ Issues já conhecidas
- ❌ Configurações inseguras do usuário (RTFM)
- ❌ Problemas que requerem acesso físico
- ❌ Problemas não relacionados

## Boas Práticas

Ao usar ParamStrike:

1. **Mantenha-o atualizado** - sempre use a versão mais recente
2. **Leia as avisos** - preste atenção aos disclaimers
3. **Use com permissão** - sempre tenha autorização para testar
4. **Siga as leis** - respeite a legislação local
5. **Relatar bugs** - nos avise se encontrar problemas
